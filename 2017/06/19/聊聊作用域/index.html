闲话少表，进入正题。

作用域是什么👻

作用域这个词其实很多 coder 都有接触过，什么全局作用局、块作用域等等，那这里的作用域有什么作用呢？

作用域（scope，或译作有效范围）是名字（name）与实体（entity）的绑定（binding）保持有效的那部分计算机程序。 ——Wikipedia

唔直接看维基的解释是不是有点头皮发麻，其实简单一点地描述就是 作用域是变量名的作用范围 ，这下大家都很明白了吧。不妨来举个🌰：

// javascript
let a = 2;

function test() {
  let a = 1;
  console.log(a);
}

test(); //1

console.log(a); //2

稍有常识的人都能看出，test函数的执行并不会影响到外部作用域的a值。当然，我们今天肯定不会讲这么显而易见的东西，我们来谈一谈 静态作用域 和 动态作用域。

没有作用域？

需要注意的是，这里探讨的作用域是在实现一门自己的语言时的一个决策问题，所以我们不妨假设我们设计了一门语言，就叫 Ylang 好了。假设我们已经完全实现了 Y，但是忘了给它实现一个作用域（也就是说 Y 的定义都在全局范围），我们来看一看会发生什么问题。假设我们有这样一段 Y 代码：

// Ylang
for i = 0; i < 10; i += 1 {
  doSomething()
}

看上去这段代码的意思是执行 doSomething 10遍，不过别忘了我们没有作用域，所以我们来实现一下 doSomething ：

// Ylang
func doSomething (): void {
  i = i == 9 ? 0 : i 
}

如果没有作用域，那很尴尬的是，最上面的那段代码将永远不会停止，很不可思议吧，但这就是没有作用域的世界。其实说是没有作用域不是很准确，因为我们确实实现了定义变量，更准确的说法是：这是个只有全局作用域的世界（好吧一样很糟糕）。

先试一试动态作用域吧

既然没有作用域很糟糕，那我们就来实现一下 Y 的作用域。看代码：

// Ylang
func doSomething(): void {
  oldi = i //先保存一手
  doOtherThing(i)
  i = oldi //再拷贝回来
}

注意到，我们这里先将 i 这个变量的原始值保存一下，然后再对 i 进行各种奇♂葩的操作，最后再把 oldi 最早保存的值给拷贝回来，这样在 doSomething 调用的前后，i 的值都没有发生变化，原先的循环也就能正常运行了。但是总是这样拷贝很头疼，不如硬点个关键字来专门干这种事请：

// Ylang
func doSomething(): void {
  let i // let 是我们定义的关键字
  doOtherThing(i)
}

这里的 let 关键字就能帮我们实现先保存最后再拷贝的功能，而我们上面实现的这种依赖拷贝来进行作用域划分 的作用域就是 动态作用域（dynamic scope）。

万事大吉了？

怎么可能嘛，明显 too young too simple，我们来看这样的一种情况：

// Ylang
variable = "foo"

func main(): void {
  let variable = "bar"
  print()
}

func print(): void {
  console.log(variable)
}

main() // bar

诶，为什么运行 main 的结果是 bar？我的全局变量 variable 明明是 foo 啊。我们来分解还原一下：

// Ylang
variable = "foo"

func main(): void {
  oldv = variable
  variable = "bar"
  print()
  variable = oldv
}

func print(): void {
  console.log(variable) // bar
}

这下问题就明确多了，在 print 函数运行的之前，variable 变量被修改为了 bar，注意我们的变量 始终保存在全局的范围里面，所以 print 打印出来的值就不符合我们的直观感受。这是动态作用域的一大问题：局部的变量定义不会对全局的变量产生影响，但是可能会对接下来调用的其他函数的局部作用域产生影响。

这个问题很讨厌，因为之后我们每次调用函数，都要考虑到在这个调用之前的每一次变量的产生和变更会不会对调用产生影响，这太过麻烦了，我们得想想别的办法。

来一手静态美滋滋

在上一部分中，我提到了这样的一个点：

我们的变量 始终保存在全局的范围里面

那如果我们把变量分块存储，在每一个括号扩起的范围内都定义一个变量和值的对照表，这样行不行呢？

从这张简单到爆炸的图来看，之前的问题很明显被解决了嘛，我们可以再定义一个关键字来帮助我们管理局部变量的对照表：

// Ylang
variable = "foo"

func main(): void {
  val variable = "bar" //硬点为 val 了
  print()
}

func print(): void {
  Console.print(variable)
}

main() // foo 结果正确啦！

这种建立多个对照表的作用域就是我们所说的静态作用域，又被成为 词法作用域（Lexical scope），现在主流的编程语言都是静态作用域的（没办法，动态有硬伤）。

看到这里有人不禁要问了：既然动态有硬伤，那我们之前探讨动态作用域干嘛呢？其实我们可以很显然得看出，动态的实现非常简单，只需要维护若干个 fallback variable 就好，但是反观静态作用域，我们还需要实现一个 hashTable，而且哈希表还会引入过多的空间复杂度，所以自己练手的话，不妨先拿简单地动态试一试。

简单的实现

说了那么多，不如简单地实现一下静态作用域：

// typescript
class LexicalScope {
  constructor(parent: LexicalScope) {
    this.parent = parent;                                            // 指向父级作用域的指针
    this.varList = Object.create(parent ? parent.varList : null);    // 变量对照表
  }

  /**
  ** 查询变量的所在作用域
  */
  private _searchVariable(name: string): LexicalScope | null {
    let scope = this;
    while(scope) {
      if(scope.varList.hasOwnProperty(name))
        return scope;
      scope = scope.parent;
    }

    return null;
  }

  /**
  ** 新建子作用域
  */
  public createSubScope(): LexicalScope {
    return new LexicalScope(this);
  }

  /**
  ** set 函数
  */
  public setVariable(name: string, value: any): any {
    let scope = this._searchVariable(name);
    if(!scope && this.parent)
      throw Error(`Undefined variable ${name}`);
    return (scope || this).varList[name] = value;
  }

  /**
  ** get 函数
  */
  public getVariable(name: string): any {
    if(name in this.varList)
      return this.varList[name];
    throw Error(`Undefined variable ${name}`);
  }

  /**
  ** 定义变量
  */
  public defVariable(name: string, value: any): any {
    return this.varList[name] = value
  }
}

上面的实现非常简单粗暴，其实我们这里实现的静态作用域是一层套着一层的，所以我利用了 JS 的原型链，很自然地形成了嵌套的结构。
